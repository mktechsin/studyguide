    Data Modeling

    Schema design & relationships · 6 min read

    A well-designed data model is the foundation of every fast, accurate Power BI report. The goal is a clean star schema with fact tables at the center and dimension tables around them.

    Star Schema

    A star schema separates your data into two types of tables:

    • Fact tables — contain measurable events (sales, orders, transactions). Wide, many rows.
    • Dimension tables — contain descriptive attributes (customers, products, dates). Narrower, fewer rows.
    💡
    Design Rule
    Aim for a single fact table per subject area. Avoid many-to-many relationships where possible — they complicate DAX and can cause double-counting.

    Relationships

    Relationships in Power BI connect tables on a common key column. They always flow from the one side (dimension) to the many side (fact).

    SettingOptionsRecommendation
    CardinalityOne-to-many, Many-to-many, One-to-oneOne-to-many preferred
    Cross-filter directionSingle, BothSingle (use Both sparingly)
    Active / InactiveActive, InactiveOnly one active path between tables
    ⚠️
    Warning
    Bidirectional cross-filtering can cause ambiguous filter paths and slow query performance. Enable it only when you have a specific reason.

    The Date Table

    Every Power BI model that uses time intelligence must have a dedicated Date table. Here's a minimal DAX Date table:

    DAX 
    Date =
VAR StartDate = DATE(2020, 1, 1)
VAR EndDate   = TODAY()
RETURN
ADDCOLUMNS(
    CALENDAR(StartDate, EndDate),
    "Year",        YEAR([Date]),
    "Month",       MONTH([Date]),
    "Month Name",  FORMAT([Date], "MMMM"),
    "Quarter",     "Q" & QUARTER([Date]),
    "Week",        WEEKNUM([Date]),
    "Day Name",    FORMAT([Date], "dddd"),
    "Is Weekend",  WEEKDAY([Date], 2) >= 6
)

    After creating the table

    1. Go to Table tools → Mark as date table and select the Date column.
    2. Create a relationship from Date[Date] to your fact table's date column.
    3. Disable Power BI's auto date/time in File → Options → Data Load.

    Calculated Tables

    Sometimes you need a helper table generated from DAX rather than loaded from a source.

    DAX 
    -- A disconnected slicer table for metric selection
Metric Selector = DATATABLE(
    "Metric Name", STRING,
    "Metric Key",  STRING,
    {
        {"Total Sales",    "sales"},
        {"Total Cost",     "cost"},
        {"Profit Margin",  "margin"}
    }
)
    🚫
    Avoid
    Don't merge tables in Power Query just to avoid relationships. A proper star schema with relationships is almost always faster and more maintainable.
