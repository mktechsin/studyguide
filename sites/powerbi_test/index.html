    Introduction to Power BI

    Getting started · 5 min read

    Power BI is Microsoft's business analytics platform that lets you connect to hundreds of data sources, create interactive reports, and share insights across your organisation.

    ℹ️
    Note
    Power BI Desktop is free to download. A Pro or Premium licence is required to publish and share reports with others.

    Key Components

    The Power BI ecosystem has three main parts you'll interact with regularly:

    • Power BI Desktop — The Windows app where you build reports and models.
    • Power BI Service — The web portal (app.powerbi.com) for sharing and collaboration.
    • Power BI Mobile — iOS and Android apps for consuming reports on the go.

    The Workflow

    Connect to your data source (Excel, SQL Server, SharePoint, APIs, etc.)

    Transform the data using Power Query to clean and shape it.

    Model by defining relationships between tables and writing DAX measures.

    Visualise by dragging fields onto a canvas and choosing the right chart types.

    Publish to the Power BI Service and share with stakeholders.

    Import vs DirectQuery

    Choosing between Import mode and DirectQuery is one of the first decisions you'll make.

    ⚠️
    Warning
    DirectQuery reports can be slow if the underlying database isn't optimised. Always benchmark before choosing this mode for large datasets.

    Import Mode

    Data is loaded into Power BI's in-memory engine (VertiPaq). Reports are fast but data is a snapshot — you need to refresh to get updates.

    DirectQuery Mode

    Every interaction queries the source database live. Always up-to-date but performance depends on your database.

    Your First M Query

    Power Query uses the M language under the hood. Here's a simple example that loads a CSV and filters it:

    M (Power Query) 
    let
    Source    = Csv.Document(File.Contents("C:\data\sales.csv"), [Delimiter=","]),
    Headers   = Table.PromoteHeaders(Source),
    Typed     = Table.TransformColumnTypes(Headers, {
                    {"Date", type date},
                    {"Amount", type number}
                }),
    Filtered  = Table.SelectRows(Typed, each [Amount] > 0)
in
    Filtered
    🚫
    Common Mistake
    Hardcoding file paths like C:\data\sales.csv will break when other users open the report. Use parameters or relative paths instead.

    Useful Keyboard Shortcuts

    Speed up your workflow in Power BI Desktop:

    • Ctrl + Z — Undo
    • Ctrl + Enter — Commit DAX formula
    • Alt + F4 — Close
    • F5 — Refresh data
