    DAX Basics

    Formulas & measures · 8 min read

    DAX (Data Analysis Expressions) is the formula language used in Power BI to create calculated columns, measures, and tables. It looks similar to Excel formulas but is designed for relational data.

    ℹ️
    Note
    Measures are calculated at query time based on filter context. Calculated columns are evaluated row-by-row when data is loaded.

    Measures vs Calculated Columns

    Measure
    A dynamic formula evaluated in the context of filters and slicers. Lives in a table but has no physical column. Use for aggregations.
    Calculated Column
    A column added to a table during data refresh. Evaluated row-by-row. Consumes memory. Use sparingly — prefer Power Query transforms.

    Essential Functions

    SUM and SUMX

    DAX 
    -- Simple aggregation
Total Sales = SUM(Sales[Amount])

-- Iterator: evaluates expression row-by-row then aggregates
Total Revenue = SUMX(
    Sales,
    Sales[Quantity] * Sales[UnitPrice]
)

    CALCULATE

    CALCULATE is the most powerful DAX function — it evaluates an expression in a modified filter context.

    DAX 
    -- Sales for a specific region, ignoring any region filter on the page
East Sales = CALCULATE(
    [Total Sales],
    Sales[Region] = "East"
)

-- Remove all filters from a column
All Products Sales = CALCULATE(
    [Total Sales],
    ALL(Products[Category])
)
    ⚠️
    Common Pitfall
    Calculated columns inside CALCULATE don't work the way you expect. Always use measures inside CALCULATE's filter arguments.

    Time Intelligence

    DAX 
    Sales YTD = TOTALYTD([Total Sales], 'Date'[Date])

Sales LY  = CALCULATE(
    [Total Sales],
    SAMEPERIODLASTYEAR('Date'[Date])
)

YoY % = DIVIDE(
    [Total Sales] - [Sales LY],
    [Sales LY],
    0   -- return 0 if denominator is blank
)
    💡
    Tip
    Time intelligence functions require a proper Date table marked as such in your model. Always build one — don't rely on auto date/time.

    DIVIDE vs /

    Always use DIVIDE(numerator, denominator, alternateResult) instead of the / operator. It safely handles division by zero and blank denominators without throwing an error.

    Variables in DAX

    Use VAR to improve readability and performance — variables are evaluated only once.

    DAX 
    Profit Margin % =
VAR TotalSales  = [Total Sales]
VAR TotalCost   = [Total Cost]
VAR Profit      = TotalSales - TotalCost
RETURN
    DIVIDE(Profit, TotalSales, 0)
