    Introduction to SQL

    Getting started · 4 min read

    SQL (Structured Query Language) is the standard language for interacting with relational databases. Whether you're querying reports, cleaning data, or building pipelines — SQL is an essential tool.

    ℹ️
    Note
    SQL syntax varies slightly between databases (PostgreSQL, MySQL, SQL Server, SQLite). These notes use standard ANSI SQL with notes for common variations.

    Basic Query Structure

    Every SQL query follows this logical order — even though you write SELECT first, the database processes clauses in a different order:

    SQL 
    SELECT   column1, column2, AGG_FUNC(column3)  -- 5. Choose columns
FROM     table_name                             -- 1. From this table
JOIN     other_table ON ...                     -- 2. Join other tables
WHERE    condition                              -- 3. Filter rows
GROUP BY column1, column2                       -- 4. Group for aggregation
HAVING   AGG_FUNC(column3) > 100               -- 5. Filter groups
ORDER BY column1 DESC                           -- 6. Sort results
LIMIT    10;                                    -- 7. Limit output

    NULL Handling

    NULL represents the absence of a value. It is not zero, empty string, or false. This catches many beginners out.

    ⚠️
    Watch out
    NULL = NULL is not TRUE in SQL — it's NULL. Always use IS NULL or IS NOT NULL to check for null values.
    SQL 
    -- Wrong
SELECT * FROM orders WHERE discount = NULL;

-- Correct
SELECT * FROM orders WHERE discount IS NULL;

-- Replace NULL with a default
SELECT COALESCE(discount, 0) AS discount FROM orders;
    💡
    Tip
    Use COALESCE(a, b, c) to return the first non-null value from a list. It's your best friend for NULL handling.
